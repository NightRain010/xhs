## 一、项目概述

本项目是一个**基于 DeepSeek 大模型的 AI Agent 应用**，核心功能是通过自然语言对话，实时查询和分析小红书平台的热点内容。

**能做什么？**

- 用户输入"今天热点是什么？"，Agent 自动调用小红书 API 获取数据并生成分析报告
- 支持搜索笔记、查看笔记详情、搜索博主、查看评论等 7 种小红书数据操作
- 支持定时邮件推送，每天自动把热点日报发到你的邮箱
- 所有对话历史持久化到 SQLite 数据库

**技术栈：**


| 层     | 技术                                           |
| ----- | -------------------------------------------- |
| 前端    | Vue 3 (Composition API) + Vite + markdown-it |
| 后端    | Python + FastAPI + SQLAlchemy (异步) + SQLite  |
| AI 模型 | DeepSeek（OpenAI 兼容接口，支持 Function Calling）    |
| 数据源   | TikHub API（小红书数据）                            |
| 定时任务  | APScheduler                                  |
| 邮件    | aiosmtplib（异步 SMTP）                          |

### 3.4 `backend/tools.py` — 小红书 API 工具（7 个）

这是 Agent 的"手"——通过 Function Calling 让 AI 自主选择调用。

#### 公共基础设施

- `_tikhub_get(path, params)` — 统一的 TikHub HTTP GET 请求封装，内置 **2 次重试**、30 秒超时、错误格式化
- `_err(data)` — 错误检测辅助函数

#### 工具列表


| #   | 函数名                                                      | 功能     | 参数                                          | 调用的 API 端点                                                               |
| --- | -------------------------------------------------------- | ------ | ------------------------------------------- | ------------------------------------------------------------------------ |
| 1   | `fetch_xhs_hot_list()`                                   | 获取热搜榜单 | 无                                           | `fetch_hot_list`                                                         |
| 2   | `search_xhs_notes(keywords, page, sort_type, note_type)` | 搜索笔记   | `keywords`（必填）、页码、排序（综合/最新/最热）、类型（全部/视频/图文） | `fetch_search_notes`                                                     |
| 3   | `fetch_xhs_note_detail(note_id)`                         | 笔记详情   | `note_id`（必填）                               | `fetch_feed_notes_v2` → `fetch_feed_notes` → `fetch_feed_notes_v4`（依次尝试） |
| 4   | `fetch_xhs_user_info(user_id)`                           | 用户资料   | `user_id`（必填）                               | `fetch_user_info_app` → `fetch_user_info`                                |
| 5   | `search_xhs_users(keywords, page)`                       | 搜索用户   | `keywords`（必填）、页码                           | `fetch_search_users`                                                     |
| 6   | `fetch_xhs_user_notes(user_id, cursor)`                  | 用户笔记列表 | `user_id`（必填）、游标                            | `fetch_home_notes_app` → `fetch_home_notes`                              |
| 7   | `fetch_xhs_note_comments(note_id, cursor)`               | 笔记评论   | `note_id`（必填）、游标                            | `fetch_note_comments`                                                    |


每个工具返回 JSON 字符串，供 Agent 作为上下文继续生成回答。

#### 工具注册表

```python
TOOL_FUNCTIONS = {
    "fetch_xhs_hot_list": fetch_xhs_hot_list,
    "search_xhs_notes": search_xhs_notes,
    ...
}
```

`XHS_TOOLS` 列表定义了 OpenAI Function Calling 格式的工具描述，Agent 根据描述决定何时调用哪个工具。

### 3.5 `backend/agent.py` — DeepSeek Agent 核心

#### 关键配置

- `client` — 使用 OpenAI SDK 连接 DeepSeek（`https://api.deepseek.com`）
- `SYSTEM_PROMPT` — 系统提示词，告诉 AI 它是小红书热点助手，有 7 种工具可用

#### 核心函数：`chat_with_agent_stream(messages)`

这是整个系统的核心——一个 **async generator**，yield SSE 事件字典。

**流式对话流程：**

```
用户消息 → DeepSeek（stream=True）
              ↓
        ┌─ 有 tool_calls？
        │   YES → 收集完整工具调用 → yield {"event": "tool_call", "name": "xxx"}
        │         → 执行工具函数 → 结果放入 messages
        │         → 循环：再次调用 DeepSeek（携带工具结果）
        │
        │   NO  → yield {"event": "delta", "content": "文本片段"}
        │         → break 跳出循环
        └─
最后 → yield {"event": "done"}
```

**SSE 事件类型：**


| 事件          | 数据                                                     | 说明        |
| ----------- | ------------------------------------------------------ | --------- |
| `delta`     | `{"event": "delta", "content": "文本片段"}`                | 流式文本，逐字推送 |
| `tool_call` | `{"event": "tool_call", "name": "fetch_xhs_hot_list"}` | 正在调用工具    |
| `done`      | `{"event": "done"}`                                    | 回复完毕      |

### 3.6 `backend/main.py` — FastAPI 主入口

#### 生命周期

```python
async def lifespan(app):
    await init_db()        # 1. 初始化数据库（建表）
    start_scheduler()      # 2. 启动 APScheduler
    await load_all_jobs()  # 3. 从 DB 加载所有已启用的邮件定时任务
    yield
    shutdown_scheduler()   # 4. 关闭时停止调度器
```

#### API 路由总览

##### 对话相关（Chat）


| 方法       | 路径                        | 说明                        |
| -------- | ------------------------- | ------------------------- |
| `POST`   | `/api/chat`               | **核心接口**，接收消息，返回 SSE 流式响应 |
| `GET`    | `/api/conversations`      | 获取所有对话列表（含消息）             |
| `GET`    | `/api/conversations/{id}` | 获取单个对话详情                  |
| `DELETE` | `/api/conversations/{id}` | 删除对话                      |


##### 邮件定时任务（Email Schedules）


| 方法       | 路径                               | 说明             |
| -------- | -------------------------------- | -------------- |
| `POST`   | `/api/email-schedules`           | 创建定时任务         |
| `GET`    | `/api/email-schedules`           | 列出所有定时任务       |
| `PUT`    | `/api/email-schedules/{id}`      | 更新定时任务（含启用/暂停） |
| `DELETE` | `/api/email-schedules/{id}`      | 删除定时任务         |
| `POST`   | `/api/email-schedules/{id}/test` | 手动触发一次测试发送     |


#### `/api/chat` 详细流程

```
1. 前端 POST { conversation_id?, message }
   ↓
2. 获取或创建 Conversation 记录
   ↓
3. 保存用户消息到 Message 表
   ↓
4. 从 DB 读取该对话的全部历史消息
   ↓
5. 构造 OpenAI 格式的 messages 数组
   ↓
6. yield SSE: {"event": "start", "conversation_id": 123}
   ↓
7. 调用 chat_with_agent_stream(messages)
   ↓
8. 逐块 yield SSE: {"event": "delta", "content": "..."} 
   （如有工具调用 yield {"event": "tool_call", ...}）
   ↓
9. 流结束 yield SSE: {"event": "done"}
   ↓
10. 拼接完整回复，保存到 Message 表
    ↓
11. commit 事务
```

### 3.7 `backend/scheduler.py` — 定时任务调度

基于 **APScheduler** 的 Cron 触发器实现。


| 函数                                 | 说明                                 |
| ---------------------------------- | ---------------------------------- |
| `start_scheduler()`                | 启动 APScheduler                     |
| `shutdown_scheduler()`             | 停止调度器                              |
| `load_all_jobs()`                  | 启动时从 DB 加载所有 `enabled=True` 的任务    |
| `add_email_job(sched)`             | 根据 EmailSchedule 记录注册/更新一个 cron 任务 |
| `remove_email_job(id)`             | 移除指定任务                             |
| `_run_agent_and_send(schedule_id)` | **任务执行函数**（被调度器触发）                 |


#### `_run_agent_and_send` 执行流程

```
1. 从 DB 读取 EmailSchedule 记录
   ↓
2. 用 schedule 的 prompt 作为用户消息，调用 chat_with_agent_stream
   ↓
3. 收集完整的 Agent 回复
   ↓
4. Markdown → HTML（调用 markdown_to_html）
   ↓
5. 通过 SMTP 发送邮件（调用 send_email）
   ↓
6. 更新 DB：last_sent_at、last_send_status
```

#### `sendMessage` 详细流程

```
1. 追加用户消息到 messages 数组（立即显示）
   ↓
2. fetch POST /api/chat（获取 ReadableStream）
   ↓
3. reader.read() 循环读取字节流
   ↓
4. TextDecoder 解码 → 按行分割 → 解析 "data: {...}" 格式
   ↓
5. 根据 event 类型处理：
   - "start"     → 记录 conversation_id，创建空的 assistant 消息
   - "tool_call" → 临时显示"正在调用工具"
   - "delta"     → 追加文本到 assistant 消息（实时渲染）
   - "done"      → 清除工具调用提示，标记流式结束
   - "error"     → 显示错误信息
   ↓
6. 流结束后刷新对话列表
```

## 六、如何启动

### 1. 配置环境变量

编辑 `backend/.env`，填入：

- `DEEPSEEK_API_KEY` — 你的 DeepSeek API Key
- `TIKHUB_API_TOKEN` — TikHub 数据接口 Token
- SMTP 相关配置（如需邮件功能）

### 2. 启动后端

```bash
cd backend
pip install -r requirements.txt
python main.py
```

后端运行在 `http://localhost:8000`，API 文档：`http://localhost:8000/docs`

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端运行在 `http://localhost:10002`

### 4. 打开浏览器

访问 `http://localhost:10002`，输入问题开始对话。

## 七、常见问题


| 问题                        | 解决方案                                 |
| ------------------------- | ------------------------------------ |
| 后端报 `AuthenticationError` | 检查 `.env` 中的 `DEEPSEEK_API_KEY` 是否正确 |
| 前端页面空白                    | 检查 npm install 是否成功，Vite 是否在运行       |
| 端口被占用                     | Vite 会自动尝试下一个端口，看终端输出的实际端口           |
| 邮件发送失败                    | 检查 SMTP 配置，QQ 邮箱需要使用授权码而非登录密码        |
| 工具调用返回错误                  | TikHub API Token 可能过期，需要更换           |
